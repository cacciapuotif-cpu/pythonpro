OPZIONE 1 - AVVIO AUTOMATICO (CONSIGLIATO)
================================================================================

3. Lo script aprirà 3 finestre:

================================================================================
TROUBLESHOOTING
================================================================================

Problema: "Porta già in uso"
Soluzione: Qualcuno sta già usando quella porta
   - Chiudi tutti i processi node.exe e python.exe dal Task Manager
   - Riavvia lo script AVVIO_LOCALE.bat

Problema: "venv not found"
Soluzione: Ambiente virtuale non trovato
   - Apri terminale in backend/
   - Esegui: python -m venv venv
   - Esegui: venv\Scripts\pip install -r requirements.txt

Problema: "npm non trovato"
Soluzione: Node.js non installato o non nel PATH
   - Verifica installazione Node.js
   - Riavvia il terminale

Problema: Frontend carica ma mostra errori 404
Soluzione: Backend non raggiungibile
   - Verifica che il backend sia in esecuzione
   - Controlla il file frontend/.env.local: REACT_APP_API_URL=http://localhost:8001

================================================================================
COMANDI UTILI
================================================================================

Vedere i log del backend in tempo reale:
  - Guarda la finestra "Backend - Gestionale"

Ricostruire il frontend (se ci sono problemi):
  - cd frontend
  - npm install
  - npm start

Backup del database:
  - docker compose exec -T backup_scheduler python run_backup.py create --type manual
  - docker compose exec -T backup_scheduler python run_backup.py list

Pulizia cache npm (se frontend non parte):
  - cd frontend
  - del /q /s node_modules
  - npm install
